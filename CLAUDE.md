## Project Overview

Ashford Wide is a Hugo-based static site for Ashford Wide, a community organization in Ashford, Surrey. Content is managed via Sveltia CMS and deployed to Cloudflare Pages (auto-built from `main`). CSS uses Tailwind v4 via PostCSS; JavaScript is vanilla (no framework).

## Commands

```bash
hugo server              # Local dev server at http://localhost:1313
npm run build            # Production build (hugo --minify)
npm run test:a11y        # Run WCAG 2.2 AA accessibility tests
npm run test:a11y:report # Open accessibility test reports in browser
```

No separate lint command — Hugo build validates templates; Tailwind errors surface during build.

## Architecture

### Content & Data

- `content/` — Markdown pages. Events and news are organized into subdirectories by year (`content/events/<year>/`, `content/news/<year>/`); also standalone pages (`about.md`, `business-directory.md`, `aed-locations.md`, `town-flags.md`, etc.) and a `content/remembrance-day/` section (e.g. `road-closures.md`).
- `data/` — YAML data files: `businesses.yaml`, `team.yaml`, `poppies/<year>.yaml`.
- Front matter is YAML. Events require `date`, `startTime` (24hr string), `endTime`, `location`; they also carry `eventStatus` and `attendanceMode` for Schema.org output.
- Permalinks strip the year from public URLs (configured in `hugo.toml`).

### Templates

- `layouts/_default/baseof.html` — single base template; all pages extend it via Hugo blocks.
- `layouts/partials/` — reusable components (`head.html`, `header.html`, `footer.html`, `opengraph.html`), plus subdirectories `jsonld/` (Schema.org partials), `icons/` (inline SVG icon partials), and `homepage/` (homepage-specific sections).
- `layouts/shortcodes/` — ~20 custom shortcodes including `paypal-donate` (and related `paypal-*` variants), `param` (reads site config values), `team`, `flag-grid`, `image`, `carousel`, `aed-map`, `pin-map`, `road-closure-map`, and `membership-tiers`.
- Event templates auto-split upcoming vs. past events by comparing dates at build time.
- Seasonal promo blocks are toggled via params in `hugo.toml`.

### Assets

- `assets/css/main.css` — Tailwind v4 entry point. Custom design tokens defined with `@theme`; custom component classes in `@layer components`. Key tokens: `--color-surface` (#212529), `--color-secondary` (#287d3d), `--color-text` (#333).
- `assets/js/` — Three scripts: `nav.js` (mobile hamburger nav), `business-directory.js` (category filtering), `vimeo-facade.js` (lazy video loading). Hugo Pipes minifies and fingerprints these in production.

### CMS

- Sveltia CMS is served at `/admin/`. Its schema lives in `static/admin/config.yml`.
- In local dev, the CMS uses the File System Access API (Chrome/Edge only) for direct file editing without a backend.
- `omit_empty_optional_fields: true` is set globally — optional front matter fields are omitted when blank rather than written as empty strings.

### SEO & Structured Data

- Schema.org JSON-LD is generated by partials for Organization, Event, NewsArticle, and WebPage types.
- Open Graph and Twitter Card tags are set in `layouts/partials/head.html`.

## Key Conventions

- Markdown filenames use `kebab-case`.
- New events go in `content/events/<year>/`; new news posts in `content/news/<year>/`.
- Tailwind utility classes are used directly in templates; avoid writing custom CSS for anything Tailwind covers.
- JS is intentionally minimal — add behaviour with vanilla JS in the existing script files rather than introducing a framework or new script files.
- Accessibility is tested automatically (`npm run test:a11y`) at WCAG 2.2 AA. Run this after any template or CSS changes.
